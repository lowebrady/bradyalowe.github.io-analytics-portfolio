06 — Statistical Analysis / R

Global Datasets —
Statistical Analysis in R

Analyzed global datasets on life expectancy, CO₂ emissions, and electricity access using R and the tidyverse ecosystem — building confidence intervals, running hypothesis tests, and creating publication-quality visualizations with ggplot2.

Role Statistical Analyst
Tools R, RStudio, tidyverse, ggplot2
Methods Confidence Intervals, Hypothesis Testing, Regression
Context MSU — Reasoning with Data
01 — Overview

From spreadsheets to statistical programming

Excel is a powerful tool for business analytics — but it has real limits when working with large datasets, complex statistical models, or reproducible research workflows. R and the tidyverse ecosystem are the answer: a programming environment built specifically for data analysis that makes complex statistics accessible and results reproducible.

This project applied core statistical methods to three global datasets — life expectancy by country, CO₂ emissions over time, and electricity access rates — to practice the full analytical workflow: clean, explore, hypothesize, test, visualize, communicate.

02 — Methods & Tools

The full statistical toolkit

  • R
    Data Wrangling with dplyr Used filter(), mutate(), group_by(), and summarize() to clean and reshape datasets. Handled missing values systematically with is.na() checks and documented all data decisions for reproducibility.
  • R
    Exploratory Data Analysis Generated summary statistics (mean, median, SD, IQR) and distribution plots for all key variables before any inferential analysis. EDA consistently surfaced the right questions to test rather than assumptions to confirm.
  • R
    Confidence Intervals Calculated 95% confidence intervals for mean life expectancy by region and for CO₂ emissions trends by income group — communicating uncertainty in estimates rather than false precision.
  • R
    Hypothesis Testing Ran two-sample t-tests comparing life expectancy between high-income and low-income countries, and chi-square tests examining the relationship between electricity access and income classification. Interpreted p-values with appropriate caution around statistical vs. practical significance.
  • R
    Visualization with ggplot2 Built publication-quality charts: scatter plots with regression lines, faceted bar charts comparing regions, time-series line plots for emissions trends, and choropleth-style heat maps for electricity access rates. Applied consistent themes and color scales throughout.
03 — Key Findings

What the global data shows

19 yrs

Life Expectancy Gap

High-income countries averaged 19 years longer life expectancy than low-income countries — a gap that narrowed significantly from 1990 to 2020 but remained large in absolute terms.

p < 0.001

Electricity ↔ Income Link

Chi-square testing confirmed a highly significant relationship between electricity access rates and income classification — access is not randomly distributed across income groups.

3 Regions

Driving Global CO₂ Growth

Three regions accounted for the majority of CO₂ emissions growth over the study period, while several regions showed flat or declining trends — important context for global climate policy discussion.

R²

GDP Explains ~70% of Variance

Simple linear regression showed GDP per capita explains approximately 70% of the variance in life expectancy across countries — leaving 30% for other factors worth investigating.

04 — What This Project Demonstrates

Skills applied

Beyond the specific findings, this project demonstrates a set of transferable analytical capabilities that apply across domains:

  • ✓
    Reproducible analysis workflow All code was written in RMarkdown, making the full analysis — from raw data to final charts — reproducible by anyone with the source files. No black boxes, no manual steps.
  • ✓
    Statistical literacy, not just computation Interpreting a p-value correctly, understanding what a confidence interval actually means, and distinguishing statistical from practical significance — these matter more than knowing which function to call.
  • ✓
    Data visualization as communication Every chart was designed to answer a specific question for a specific audience — not to display data for its own sake. Axes are labeled clearly, color is used purposefully, and chart types match the data structures being shown.
